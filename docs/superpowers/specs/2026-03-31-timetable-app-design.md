# 웹 기반 학교 시간표 및 스터디 플래너 설계 문서 (Design Spec)

## 1. 개요 (Overview)
사용자가 학교 시간표를 등록하고, 각 과목별 시험 일정(D-Day) 및 공부 계획(To-Do)을 세우며 성취도를 관리할 수 있도록 돕는 풀스택 웹 애플리케이션입니다. 메인 대시보드의 달성률 차트와 D-Day 표시를 통해 지속적인 학습 동기를 제공하는 것이 핵심입니다.

## 2. 레이아웃 구조 (Layout)
- **좌측 사이드바 (Sidebar Menu)**: 로그인된 사용자 프로필, 대시보드, 시간표 관리, 플래너 메뉴 등 주요 페이지간 이동 토글
- **우측 메인 컨텐츠 영역 (Main Content View)**: 각 선택된 메뉴의 세부 내용이 렌더링되는 넓은 뷰포트 영역

## 3. 기능 및 사용자 흐름 (Core Features & User Journey)

### 3.1. 온보딩 및 회원 관리
- **가입 및 로그인 절차**: 사용자 계정 생성, 안전한 데이터베이스 연동.
- **프로필 생성**: 이름, 소속, 각오 등을 기입.

### 3.2. 메인 대시보드 (동기부여 중심)
- **달성률 시각화**: 전체 과목 기준으로 '이번 주/달 완수한 투두 리스트 비율'을 예쁜 원형 혹은 막대 차트로 표시.
- **시험 D-Day 카드**: 가장 임박한 시험일정, 과목명, 남은 일수 강조.

### 3.3. 시간표 및 과목/시험 세팅
- **과목 등록**: 이번 학기 과목 이름 등록 (색상 태그).
- **주간 시간표 확인**: 과목별 주간 스케줄 확인.
- **일정(시험) 등록**: 개별 과목의 시험 혹은 중요한 과제 제출 일정을 등록.

### 3.4. 스터디 플래너 (과목별 체크리스트)
- **투두(To-Do) 추가**: 각 과목별로 '해야 할 일' 리스트를 생성.
- **상태 관리 체계**: 미완료 상태에서 체크 박스 틱(Tick)을 하면 '완료' 처리.
- **필터링 기능**: 스위치나 탭을 이용해 "전체 보기", "진행 중(미완료)", "완료" 항목을 구분하여 출력.

## 4. 백엔드 및 데이터 구조 (Architecture Details)
- **프론트엔드 스택**: React / Vite 활용 (혹은 순수한 HTML/CSS/JS 조합). 사이드바 및 동적인 컴포넌트 처리에 최적.
- **백엔드 스택**: Node.js + Express
- **데이터 저장소**: SQLite 또는 PostgreSQL 사용. (회원 테이블, 과목 테이블, 일정 테이블, 투두 테이블간 릴레이션 관계 형성)

## 5. 예외 및 에러 처리 방침
- 미로그인 사용자의 접근은 로그인 페이지로 자동 로드 (비인가 에러 핸들링).
- 학습 계획이 비어있을 땐 비어있다는 일러스트와 함께 "계획 세우러 가기" 버튼을 노출(Empty State Design).
